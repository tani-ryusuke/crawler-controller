# クローラーロボット開発プロジェクト（コントローラー・送信機側）

本リポジトリは、ESP32-S3マイコンおよびESP-IDF（v5.3.5）開発環境を用いた、カスタムワイヤレスコントローラー（送信機）のメインファームウェアです。アーケードレバーと8個の物理ボタン（計12入力）を搭載し、FreeRTOSによる20ms（50Hz）の定周期高速ポーリングと、ESP-NOWを用いた低遅延無線通信を統合しています。特に、操縦者の利便性とデバッグ性を考慮し、手元のスマートLED（WS2812）で通信品質をリアルタイムに可視化する独自のテレメトリ機能を備えています。

## 主な実装機能と技術的特徴

### 1. 簡易移動平均を用いた通信品質（RSSI代替）のリアルタイムLED可視化
ESP-NOWの送信完了コールバック（`on_data_sent`）を活用し、送信の成否に応じて増減する独自の通信スコア（履歴重み：100）を用いた簡易移動平均を算出。
* **技術的工夫**: 算出された成功率（0.0〜1.0）を基に、RGBのカラー輝度を滑らかに動的合成。通信が極めて安定していれば「緑」、パケットロスが発生し始めると「黄色」を経て「赤」へとリアルタイムにグラデーション変化します。これにより、実機運用時における電波環境の悪化やパケットこぼれを、ハードウェア単体で直感的にデバッグ・モニタリング可能です。

### 2. 格闘ゲーム準拠の「テンキー方向デコード（1〜9方式）」ロジック
アーケードレバーの上下左右（4ピン）入力を個別に送信するのではなく、内部ソフトウェアで斜め入力の同時押しまで含めた「テンキー表記（1〜9の数値）」へリアルタイムにデコードして送信。
* **メリット**: 無線通信パケットのペイロードサイズを最小限に削ぎ落とし、通信の軽量化・低遅延化を徹底しています。本体側での「隠しコマンド入力」といった複雑なアクション判定への拡張性も確保しています。

### 3. ビットマスクによる効率的なGPIO一括初期化
レバー4ピン、ボタン8ピンの計12個におよぶGPIOピンを1つずつ個別に初期化する無駄を省き、ビット演算（`1ULL`）を用いた一括マスク登録を実装。
* **構造の利点**: `gpio_config_t` 構造体を用いて1回の設定投入で全ピンの「入力モード」「内部プルアップ有効（アクティブロー）」を確実かつクリーンに初期化。可読性とメンテナンス性に優れたコード構造に仕上げています。

### 4. RMT周辺機器を用いたWS2812スマートLEDの精密制御
1個のスマートLED（WS2812）を正確に発光させるため、ESP32-S3のハードウェア機能であるRMT（Remote Control Peripheral）を駆動。10MHzの精密な分解能（`resolution_hz`）を設定し、タイミングシビアなシリアルLED制御を安定して実行しています。

### 5.堅牢なNVS初期化・エラーリカバリ
Wi-Fiや通信スタックの起動に必須となるNVS（Non-Volatile Storage）フラッシュメモリの初期化において、ページの枯渇（`ESP_ERR_NVS_NO_FREE_PAGES`）やバージョン不一致（`ESP_ERR_NVS_NEW_VERSION_FOUND`）を検知した場合に、自動で領域を強制消去（Erase）して再初期化を試みるリカバリロジックを実装。意図しないデータ不整合による起動スタックを防ぐ、セーフティ設計を取り入れています。
